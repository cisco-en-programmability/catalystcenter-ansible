.. Notes

Notes
-----

.. note::
   - SDK Method used are topology.Topology.get\_topology\_details\_v1,
   - Paths used are get /dna/intent/api/v1/topology/l2/{vlanID},
   - It should be noted that this module is an alias of topology\_layer\_2\_v1\_info
   - Supports \ :literal:`check\_mode`\ 
   - The plugin runs on the control node and does not use any ansible connection plugins, but instead the embedded connection manager from Cisco CATALYST SDK
   - The parameters starting with catalystCenter\_ are used by the Cisco CATALYST Python SDK to establish the connection

        password: "{{password}}"
        verify: "{{verify}}"
        api_port: "{{api_port}}"
        version: "{{version}}"
        debug: "{{debug}}"
        headers: "{{my_headers | from_json}}"
        vlanID: string
      register: result
